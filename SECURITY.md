# Политика безопасности

## Сообщение об уязвимостях

Если вы нашли уязвимость, **не открывайте публичный issue**. Сообщите приватно через
GitHub Security Advisories (вкладка «Security» репозитория) или на контактный адрес сопровождающих.

Пожалуйста, приложите:

- описание проблемы и потенциального воздействия;
- шаги воспроизведения (PoC по возможности);
- затронутые версии/компоненты (`packages/*`, `runtime/*`, плагины).

Мы подтвердим получение и будем держать вас в курсе процесса устранения.

## Область

Особое внимание — к границе доверия моста JS ↔ native: проверка источника сообщений, разрешения/области
(scopes), валидация аргументов (см. [docs/architecture/bridge.md](docs/architecture/bridge.md) §9).
