# ADR-001: Архитектура Runtime — нативный контейнер C++/QML

- **Статус:** Accepted
- **Дата:** 2026-06-27
- **Связанные требования:** FR-R1…R10, NFR-3, NFR-7
- **Связанные ADR:** ADR-002 (Bridge), ADR-004 (движки), ADR-007 (сборка)

## Контекст
ОС Аврора исполняет приложения как нативные (C++ + QML), собираемые Aurora SDK и упаковываемые в RPM.
Веб-приложение отображается через фреймворк **Webview (Chromium)** на базе CEF (`ru.auroraos.webview`) —
рекомендуемый для гибридных веб-приложений путь (есть и легаси Webview на Gecko, см. [ADR-004](ADR-004-webview-engine-abstraction.md)).
Нужен контейнер, который запускает WebView, обслуживает мост, lifecycle, навигацию, splash, ресурсы,
разрешения, deep links — единообразно для любого веб-приложения.

## Рассмотренные варианты
1. **Тонкий нативный контейнер на C++/QML (выбран).** Минимальный нативный «хост», вся бизнес-логика —
   в вебе и плагинах. По духу — Capacitor/Tauri (WebView + runtime + плагины).
2. **«Толстый» нативный каркас с UI на QML.** Часть UI/логики в QML. Отвергнут: противоречит цели
   «писать на вебе», увеличивает порог входа.
3. **Только инструментарий-генератор без рантайма.** Каждый проект генерирует свой контейнер с нуля.
   Отвергнут: нет переиспользуемого, тестируемого ядра; дублирование и расхождения.

## Решение
Runtime — **тонкий нативный контейнер (C++/QML)**, поставляемый как шаблон/библиотека, который
Build System параметризует конфигом и набором плагинов. Контейнер владеет WebView (через абстракцию
движка, ADR-004) и native-стороной моста (ADR-002), но не содержит прикладной логики.

## Последствия
- (+) Низкий порог входа: разработчик пишет веб, не трогает C++/QML.
- (+) Переиспользуемое, тестируемое ядро; единообразие приложений.
- (+) Чёткое разделение ответственности (контейнер ↔ плагины ↔ веб).
- (−) Контейнер обязан скрывать различия движков/версий Аврора (сложность в `runtime/engines/`).
- (−) Возможности ограничены тем, что предоставляют плагины (но это и есть модель расширения).

## Заметки/верификация
Точные имена пакетов/модулей WebView Аврора (`ru.auroraos.webview`, `pkgconfig(aurorawebview)`, `libcef`),
порядок инициализации окна (ApplicationWindow/pageStack) и интеграция splash — проверить на актуальном SDK.
