# ADR-003: Plugin API — манифест как источник истины + статическая регистрация + кодогенерация

- **Статус:** Accepted
- **Дата:** 2026-06-27
- **Связанные требования:** FR-P1…P7, FR-S1…S3, NFR-4, NFR-5
- **Связанные ADR:** ADR-002 (Bridge), ADR-006 (формат), ADR-008 (кодоген)

## Контекст
Главный недостаток Cordova — ручная регистрация плагинов (правка нативных конфигов) и дублирование
JS-обёрток. Нужна модель, где добавление плагина не требует ручной правки нативного проекта, а слои
(native-регистрация, JS-обёртка, типы, доки) не расходятся.

## Рассмотренные варианты
1. **Манифест-центричная модель + кодоген + статическая регистрация на сборке (выбран).** Плагин
   описывается манифестом (SoT); из него генерируются JS-обёртка, TS-типы, native-регистрация и доки;
   нативные части линкуются на этапе сборки.
2. **Императивная регистрация в коде (как ранний Capacitor/Cordova).** Отвергнут: дублирование,
   расхождение слоёв, ручная рутина.
3. **Динамическая загрузка нативного кода плагинов в рантайме.** Отвергнут: риски безопасности,
   сложность на Аврора (RPM/подписи), непредсказуемость; статическая модель безопаснее.

## Решение
Плагин = **манифест (SoT)** + нативная реализация (по контракту Native SDK) + **сгенерированные**
JS-обёртка и типы. Регистрация **двухфазная**: build-time (генерация реестра, включение nativeDeps,
агрегация разрешений) + runtime (инстанцирование и публикация API). Совместимость плагина
(Runtime/протокол/ОС/движок) объявляется в манифесте и проверяется при загрузке.

## Последствия
- (+) Нулевая ручная регистрация; нет дублирования слоёв; типобезопасность.
- (+) Безопасность и предсказуемость (нет динамического native-кода).
- (+) Независимая эволюция плагинов (semver + объявленная совместимость).
- (−) Требуется надёжный кодоген и валидатор манифеста (ADR-008).
- (−) Новый нативный плагин всё же требует пересборки (приемлемо: native нельзя «горячо» менять).

## Заметки/верификация
Формат манифеста финализируется в ADR-006. Контракт conformance-тестов (FR-T1) — отдельной спекой.
