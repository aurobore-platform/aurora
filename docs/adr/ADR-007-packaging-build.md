# ADR-007: Упаковка и сборка — CMake + RPM через Aurora SDK

- **Статус:** Accepted
- **Дата:** 2026-06-27
- **Связанные требования:** FR-CF2, FR-CF3, NFR-3, NFR-11
- **Связанные ADR:** ADR-001 (Runtime), ADR-005 (CLI)

## Контекст
Приложения ОС Аврора собираются Aurora SDK и упаковываются в **RPM** (`.spec`, rpmbuild), с `.desktop`
(метаданные/разрешения/иконки). Поддерживаются системы сборки **qmake** и **CMake**; исследование
(в т.ч. опыт сообщества) показывает, что **CMake гибче** и лучше подходит для нетривиальных зависимостей
(поиск библиотек, RPATH, точные RPM-имена вместо ненадёжного pkgconfig).

## Рассмотренные варианты
1. **Генерировать CMake-проект + `.spec`/`.desktop`, собирать штатным Aurora SDK (выбран).**
   Используем родной путь Аврора; CMake даёт контроль над зависимостями плагинов.
2. **qmake вместо CMake.** Проще для простых случаев, но скрывает сложность и упирается в ограничения
   при нетривиальных зависимостях (вывод из исследования).
3. **Своя система сборки/упаковки в обход Aurora SDK.** Отвергнут: несовместимость со стором/подписями
   Аврора, дублирование, хрупкость.

## Решение
Build System **детерминированно генерирует** нативный проект Аврора на **CMake** с `.spec` и `.desktop`
и вызывает **штатную сборку Aurora SDK** (CMake → rpmbuild → `.rpm`). Нативные зависимости плагинов
(`nativeDeps`) проецируются в `BuildRequires`/`Requires` и зависимости CMake; библиотеки ищутся через
`find_package`/pkgconfig с приоритетом точных RPM-имён; учитываются проблемы RPATH/зависимостей WebView.
Разрешения и deep-link схемы проецируются в `.desktop`.

## Последствия
- (+) Совместимость со стандартным конвейером Аврора (стор, подписи).
- (+) Контроль зависимостей плагинов; воспроизводимость (NFR-11).
- (+) Гибкость CMake для разных движков/версий (NFR-3).
- (−) Требуется Aurora SDK в окружении сборки (проверяет `doctor`).
- (−) Сложность генератора нативного проекта (изолирована в `@aurobore/build`).

## Заметки/верификация
**Подтверждено** (см. [aurora/verification-status.md](../aurora/verification-status.md)):
`pkgconfig(aurorawebview)` (C-3), наличие `libcef` (C-6), RPM-упаковка/обязательная подпись (C-8),
архитектуры/имя таргета (C-10).

**Остаётся проверить на SDK** (реестр §2): расхождение точного имени рантайм-RPM
`ru.auroraos.webview` vs `ru.aurora.webview` и имя `-devel`-пакета (**V-2**); необходимость бандла
`libcef` и настроек RPATH `$ORIGIN/../lib/cef` (**V-3**); точный синтаксис разрешений/схем в `.desktop` (**V-6**).
Зафиксировать матрицу таргетов/архитектур.
