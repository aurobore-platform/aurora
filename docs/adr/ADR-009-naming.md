# ADR-009: Кодовое имя проекта и неймспейсы

- **Статус:** Accepted
- **Дата:** 2026-06-27
- **Связанные требования:** NFR-10
- **Связанные ADR:** ADR-007 (упаковка/идентификаторы Аврора)

## Контекст
Проекту нужно единое имя для пакетов, CLI, глобального JS-объекта, безопасной схемы ресурсов и
документации. В исходной постановке (`task.txt`) использовалось описательное `aurora-platform`.
Требования к имени: узнаваемо и связано с «Авророй»; **не конфликтует** с официальным брендом и
неймспейсом ОС Аврора/Aurora SDK (`ru.auroraos.*`); пригодно как npm-scope и идентификатор схемы;
свободно в npm и доменах (требует юридической/доступностной проверки).

## Рассмотренные варианты
1. **Aurobore (выбран).** Слияние «Aurora + Borealis»: связано с Авророй, короткое, бренд-подобное
   (как Capacitor/Tauri). Глобал `Aurobore`, scope `@aurobore/*`, CLI `aurobore` (алиас `bor`),
   схема `aurobore-app://`.
2. **Borealis.** Образно связано с Авророй (aurora borealis), но менее явная продуктовая связь и выше
   риск пересечения с существующими проектами/брендами «Borealis».
3. **aurora-platform / aurora-runtime.** Описательно, но рискует пересечься с официальным брендом
   «Аврора» и инструментами SDK; менее «продуктовое».
4. **Polaris / иное.** Нейтрально, но слабее тематическая связь.

## Решение
Использовать кодовое имя **Aurobore**:

| Сущность | Значение |
|---|---|
| Глобальный JS-объект | `Aurobore` |
| npm-scope | `@aurobore/*` (`@aurobore/core`, `@aurobore/cli`, …) |
| CLI | `aurobore`, алиас `bor` |
| Логическое имя схемы локальных ресурсов | `aurobore-app://` (конфиг/доки; wire в M1 — loopback HTTP, см. V-13) |
| Файл конфигурации проекта | `aurobore.config` |

Имя принимается как **рабочее кодовое имя** проекта. Финальная юридическая проверка (товарные знаки) и
проверка доступности `@aurobore` в npm/доменах выполняется до публичного релиза (этап RC); при конфликте
имя меняется новым ADR **без изменения архитектуры**.

> Важно: имя пакета **приложения пользователя** в ОС Аврора (RPM/appId, например `ru.<org>.<App>`) —
> отдельная сущность из мира Аврора и **не** связана с npm-scope `@aurobore` (см.
> [aurora/glossary.md](../aurora/glossary.md), [ADR-007](ADR-007-packaging-build.md)). Неймспейс Аврора —
> `ru.auroraos.*`; неймспейс наших npm-пакетов — `@aurobore/*`; пересечений нет.

## Последствия
- (+) Единое узнаваемое имя во всей документации, пакетах, CLI и схеме ресурсов уже на этапе проектирования.
- (+) Не пересекается с официальным неймспейсом Аврора `ru.auroraos.*`.
- (−) Остаётся формальный риск переименования перед релизом (юр. проверка) — принят осознанно, финализация на RC.
- Действие (до RC): проверить занятость `@aurobore` в npm, домен и товарные конфликты.

## Заметки/верификация
Проверить занятость scope `@aurobore` в npm, доступность домена и совпадения с существующими
проектами/брендами. Это коммерческо-юридическая проверка, **не влияющая на архитектуру**.
