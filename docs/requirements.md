# Функциональные и нефункциональные требования

Требования разделены по приоритету по системе **MoSCoW**:

- **MUST** — обязательно (без этого продукт не имеет смысла; в основном цель MVP/1.0).
- **SHOULD** — желательно (важно, но может прийти после MVP).
- **COULD** — дополнительно (ценные идеи, кандидаты на пост-1.0).

Каждое требование имеет идентификатор (`FR-*` функциональные, `NFR-*` нефункциональные)
для трассируемости с архитектурой, дорожной картой и чек-листами.

---

## 1. Обязательные возможности (MUST)

### Runtime
- **FR-R1.** Запускать нативное приложение Аврора, открывающее WebView во весь экран.
- **FR-R2.** Загружать локальное веб-приложение (HTML/CSS/JS) из пакета приложения.
- **FR-R3.** Поддерживать SPA-навигацию (history API) и корректную обработку аппаратной кнопки «назад».
- **FR-R4.** Показывать splash screen до готовности веб-приложения.
- **FR-R5.** Прокидывать события жизненного цикла приложения (pause/resume/foreground/background) в JS.
- **FR-R6.** Отдавать локальные ресурсы без сырого `file://` для entry (M1: loopback HTTP origin +
  логический ключ `aurobore-app://localhost/…`; см. V-13).
- **FR-R7.** Работать на **Chromium/CEF-WebView** актуальной линейки Аврора (база 5.1.5/5.1.6+).
  Транспорт моста — на WebView async API (`sendAsyncMessage`/`onRecvAsyncMessage`/`runJavaScript`,
  `ru.auroraos.WebView`), изолирован за тонким внутренним швом (без поддержки Gecko).
  CEF `CefMessageRouter`/`cefQuery` — деталь реализации ниже. См. [ADR-004](adr/ADR-004-webview-engine-abstraction.md).

### Bridge
- **FR-B1.** Вызов нативного метода из JS с аргументами и возвратом результата как **Promise**.
- **FR-B2.** Корреляция «запрос ↔ ответ» (уникальные идентификаторы вызовов).
- **FR-B3.** Передача структурированных данных (JSON-сериализуемых) в обе стороны.
- **FR-B4.** Доставка ошибок native → JS как отклонение Promise со структурой ошибки (код, сообщение, данные).
- **FR-B5.** Двунаправленные события (native → JS и JS → native) по модели подписки.
- **FR-B6.** Стримы (многократная доставка по одной подписке) для геолокации/сенсоров/прогресса.

### Plugin System
- **FR-P1.** Плагин описывается **манифестом** (методы, события, типы, разрешения, нативные зависимости).
- **FR-P2.** Автоматическое **обнаружение и регистрация** установленных плагинов при старте Runtime
  (без ручной правки нативных конфигов).
- **FR-P3.** Экспорт API плагинов в JS как `Aurobore.<Plugin>.<method>()` и через npm-обёртки.
- **FR-P4.** Версионирование плагина и проверка совместимости с версией Runtime/Bridge.
- **FR-P5.** Набор стандартных плагинов MVP: **Device, Storage (key-value), FileSystem, Clipboard, Network**.

### CLI
- **FR-C1.** `create` — создание проекта из шаблона.
- **FR-C2.** `dev` — запуск dev-сервера с live reload.
- **FR-C3.** `build` — сборка пакета приложения под Аврора.
- **FR-C4.** `run` — запуск на эмуляторе/устройстве.
- **FR-C5.** `plugin add|remove|list` — управление плагинами.
- **FR-C6.** `doctor` — диагностика окружения (Aurora SDK, зависимости, конфиг).

### Configuration & Build
- **FR-CF1.** Единый файл конфигурации проекта (`aurobore.config`) с appId, именем, версией,
  ориентацией, разрешениями, списком плагинов.
- **FR-CF2.** CLI **генерирует** нативный проект Аврора (структуру C++/QML, CMake, `.spec`, `.desktop`,
  иконки) из конфига и установленных плагинов; пользователь не правит их вручную.
- **FR-CF3.** Build копирует веб-сборку, подключает плагины, вызывает штатную сборку Аврора, выдаёт пакет.

### TypeScript SDK
- **FR-S1.** Публичный типизированный API (`@aurobore/core` + пакеты плагинов).
- **FR-S2.** Типы и JS-обёртки **генерируются из манифестов** плагинов (без ручного дублирования).
- **FR-S3.** Единая модель ошибок и событий в SDK.

### Документация и экосистема
- **FR-D1.** Документация: создание проекта, написание плагина, API, архитектура моста, публикация.
- **FR-D2.** Минимум один официальный шаблон (vanilla) и один рабочий пример.

## 2. Желательные возможности (SHOULD)

- **FR-R8.** Deep links / обработка кастомных URI-схем.
- **FR-R9.** Гранулярная модель разрешений и областей (scopes) на уровне моста.
- **FR-R11.** Системный chrome и safe-area **из коробки (zero-config):** runtime автоматически учитывает
  insets status bar / cutout / клавиатуры (native → CSS vars + встроенный `aurobore-chrome.css`);
  приложение из шаблона корректно отображается без ручного `env(safe-area-inset-*)` в коде разработчика;
  WebView fullscreen (фикс. геометрия), клавиатура overlay через `KeyboardInput` + CSS inset (см. [alpha-plan](alpha-plan.md) A2, [immersive-ui](tutorials/immersive-ui.md)).
- **FR-B7.** Передача бинарных данных эффективно (через blob/ArrayBuffer или ссылки на ресурсы, не base64 в JSON).
- **FR-B8.** Батчинг событий и backpressure для высокочастотных стримов.
- **FR-P6.** Расширенный набор плагинов: **Camera, Geolocation, Notifications (local), Share, Sensors**.
- **FR-P7.** Установка сторонних плагинов из npm и автоинтеграция их нативной части.
- **FR-C7.** `dev` с **Hot Reload** (HMR) поверх live reload.
- **FR-C12.** `dev --web` — browser mock mode: приложение в desktop-браузере с loopback-мостом и
  детерминированными mock-ответами встроенных плагинов; без эмулятора/SDK. См. [web-mock-mode.md](dev/web-mock-mode.md).
- **FR-C8.** `plugin create` — генерация скелета нового плагина из манифеста.
- **FR-C9.** `publish` — публикация приложения/плагина (в стор Аврора / npm).
- **FR-S4.** Официальные шаблоны для React / Vue / Svelte.
- **FR-S6.** Опциональные полифилы W3C API (`@aurobore/polyfills`): `navigator.geolocation`,
  `navigator.share`, `Notification`, `navigator.clipboard`, `navigator.mediaDevices.getUserMedia`
  → плагины Aurobore; opt-in через `web.polyfills` в конфиге. См. [w3c-polyfills.md](dev/w3c-polyfills.md),
  [ADR-010](adr/ADR-010-w3c-polyfills.md).
- **FR-D3.** DevTools: лог bridge-вызовов, инспекция, базовое профилирование.
- **FR-D5.** JSON Schema для `aurobore.config` и `plugin.manifest` в `@aurobore/build`; шаблоны с `$schema`
  для автодополнения в редакторе. См. [configuration.md](architecture/configuration.md) §7.
- **FR-T1.** Набор тестов совместимости плагинов (conformance suite).
- **FR-T2.** E2E device harness: сквозной сценарий на эмуляторе (`pnpm e2e:verify`). См. [e2e-testing.md](dev/e2e-testing.md).

## 3. Дополнительные идеи (COULD)

- **FR-P8.** Плагины: **Contacts, Biometric, Secure Storage, Push Notifications, Background Tasks**.
- **FR-R10.** Многооконность / несколько WebView, фоновые сервисы.
- **FR-B9.** Бинарный/более компактный протокол моста для горячих путей.
- **FR-C10.** `migrate` — миграция конфигурации/проекта между версиями платформы.
- **FR-C11.** Плагин-маркетплейс и индекс совместимости.
- **FR-S5.** Адаптеры/биндинги к популярным фреймворкам (хуки React, composables Vue).
- **FR-D4.** Автогенерация справочника API из манифестов в сайт документации.
- **FR-X1.** Кросс-платформенный режим (повторное использование плагинов на других ОС) — стратегическая опция.
- **FR-R13.** OTA-обновления веб-бандла: проверка подписи, скачивание, атомарное применение, откат;
  события `update:*` в JS. Только веб-часть; натив/permissions — через RPM. См. [ADR-012](adr/ADR-012-ota-live-updates.md).
- **FR-C14.** CLI `aurobore update keygen|publish|list|rollback` — формирование подписанного OTA-артефакта
  и управление каналами. См. [ADR-012](adr/ADR-012-ota-live-updates.md).

## 4. Нефункциональные требования (NFR)

| ID | Категория | Требование |
|---|---|---|
| **NFR-1** | Производительность | Накладные расходы одного bridge-вызова — минимальны; стримы не блокируют UI-поток. |
| **NFR-2** | Безопасность | Нет произвольного доступа к нативу из веба; вызовы ограничены разрешениями/областями; ресурсы — через безопасную схему; проверка источника сообщений моста. |
| **NFR-3** | Совместимость | Работа на Chromium/CEF-WebView поддерживаемой линейки Аврора (5.1.5/5.1.6+, 5.2.x); явная матрица поддержки версий ОС. Gecko вне области поддержки. |
| **NFR-4** | Расширяемость | Новый плагин добавляется без изменения ядра Runtime/Bridge. |
| **NFR-5** | Эволюционируемость | Версионируются: конфиг, протокол моста, API плагинов, API Runtime; правила обратной совместимости. |
| **NFR-6** | DX (опыт разработчика) | От `create` до запущенного приложения — минимум шагов; внятные ошибки; `doctor` чинит окружение. |
| **NFR-7** | Надёжность | Падение/исключение в плагине не роняет Runtime; ошибки доходят до JS как отклонённый Promise. |
| **NFR-8** | Тестируемость | Компоненты тестируются изолированно; есть conformance-тесты для плагинов и моста. |
| **NFR-9** | Документированность | Каждое публичное API и формат — документированы; решения — в ADR. |
| **NFR-10** | Открытость | Лицензия для open-source; прозрачный процесс RFC/ADR; вклад сообщества. |
| **NFR-11** | Воспроизводимость сборки | Детерминированная генерация нативного проекта и пакета из конфига. |
| **NFR-12** | Наблюдаемость | Логирование моста и Runtime, диагностика, коды ошибок с пространствами имён. |

## 5. Ограничения и допущения

- **A1.** Целевая платформа — ОС Аврора; сборка использует **Aurora SDK** (требуется в окружении сборки).
- **A2.** Целевой движок — **Chromium/CEF**. Подтверждено офиц. документацией: Webview (Chromium)
  доступен с **5.1.3**, Gecko (легаси) — вне поддержки (см. [ADR-004](adr/ADR-004-webview-engine-abstraction.md),
  [aurora/webview.md](aurora/webview.md)).
- **A3.** Разрешения декларируются в артефактах Аврора (`.desktop`) и должны выводиться из конфига
  (подтверждено: [aurora/sandbox-and-permissions.md](aurora/sandbox-and-permissions.md)).
- **A4.** Упаковка — **RPM** (`.spec`); нативная сборка — **CMake** (предпочтительно перед qmake).
- **A5.** Имена/модули WebView **частично подтверждены** официально (`pkgconfig(aurorawebview)`,
  QML `ru.auroraos.WebView`, C++ `Aurora::WebView::*`, `libcef`). Открыто: точное имя рантайм-RPM
  (`ru.auroraos.webview` vs `ru.aurora.webview`) и `-devel`. Статус и план проверки —
  [aurora/verification-status.md](aurora/verification-status.md).

## 6. Критерии приёмки MVP

MVP считается достигнутым, когда выполнены: FR-R1…R7, FR-B1…B6, FR-P1…P5, FR-C1…C6, FR-CF1…CF3,
FR-S1…S3, FR-D1…D2 и NFR-1, 2, 6, 7. Подробнее — в [дорожной карте](roadmap.md) и [чек-листах](checklists.md).
