# Песочница и разрешения в ОС Аврора

> Источник: [`/doc/sdk/app_development/work/sandbox`](https://developer.auroraos.ru/doc/sdk/app_development/work/sandbox).
> Версия документации: **ОС Аврора 5.2.1**.

## 1. Модель изоляции

В ОС Аврора реализован механизм безопасного исполнения — **изоляция (песочница)**. Каждое приложение
запускается в изолированном окружении, ограничивающем доступ к API и данным.

- Песочница реализована через оболочку **Firejail**: процесс приложения и все его потомки получают
  собственное представление глобально разделяемых ресурсов ядра (сетевой стек, таблица процессов,
  таблица монтирования).
- Цель — минимизировать риск компрометации системы при запуске недоверенного/уязвимого ПО.

## 2. Четыре следствия для приложения

Из документации, при работе в песочнице приложение обязано:

1. **Запрашивать разрешения** на доступ к части API и каталогов — они **декларируются в `.desktop`-файле**
   (требуемые функции для доступа к типам данных, к отдельным каталогам, к API, взаимодействующим с
   другими приложениями/системными сервисами).
2. **Работать с пользовательскими каталогами через механизм общих директорий** — прямого доступа к
   произвольным файлам других пакетов нет; обмен данными — через «общие директории».
3. **Декларировать собственную D-Bus-службу** в `.desktop`-файле. На основе значений
   **`OrganizationName`** и **`ApplicationName`** приложению выдаются права на владение именем службы
   D-Bus на сессионной шине.
4. **Учитывать особенности отладки** изолированных приложений: через `gdb` или утилиту
   `runtime-manager-tool`.

## 3. Где живут разрешения

Разрешения — это **декларация в `.desktop`-файле**, а не вызов в рантайме. То есть «право на камеру/гео/
сеть/каталог» фиксируется на этапе упаковки. Это напрямую влияет на модель Aurobore:

- В нашем глоссарии **Permission** «проецируется в артефакты Аврора (`.desktop`)» —
  это согласуется с официальной моделью ([glossary.md](../glossary.md)).
- Значит, секция разрешений в `aurobore.config` должна **детерминированно** генерировать корректные
  записи в `.desktop` (см. [architecture/configuration.md](../architecture/configuration.md)).

## 4. Проекция на Aurobore

| Аспект песочницы | Что должен делать Aurobore | Документ |
|---|---|---|
| Разрешения в `.desktop` | Генерировать декларации из конфига | [architecture/configuration.md](../architecture/configuration.md) |
| Общие директории / каталог данных | Asset Loader и FS-плагин работают в пределах разрешённых каталогов | [architecture/runtime.md](../architecture/runtime.md), [plugins/standard-plugins.md](../plugins/standard-plugins.md) |
| D-Bus (`OrganizationName`/`ApplicationName`) | Корректно задавать имена при генерации проекта | [adr/ADR-009](../adr/ADR-009-naming.md) |
| Отладка изолированного приложения | Учесть в DevTools/диагностике (`doctor`) | [architecture/cli.md](../architecture/cli.md) |
