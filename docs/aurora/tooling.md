# Инструменты разработки ОС Аврора

> Источник: [`/doc/sdk/tools`](https://developer.auroraos.ru/doc/sdk/tools). Версия: **ОС Аврора 5.2.1**.
> Назначение — знать штатный инструментарий, который Aurobore **оркеструет** (а не заменяет), и на что
> опираются `doctor`/CI.

## 1. Перечень (по официальной классификации)

| Категория | Инструмент | Назначение | Связь с Aurobore |
|---|---|---|---|
| **Сборка** | `mb2` | сборка RPM и управление зависимостями (Build Engine) | вызывается `aurobore build`/`run` |
| | `apptool` | кросс-сборка пакетов (вариант BT), независим от `mb2`/`sb2` | альтернативный путь сборки |
| | `sb2` (Scratchbox2) | модуль кросс-компиляции | низкий уровень сборки/диагностики |
| | Linaro GCC | компилятор (preprocess/compile/link, отладочная информация) | сборка нативной части |
| **Отладка** | `ADT` | консольная утилита отладки/управления/взаимодействия с ОС | основа для `aurobore run`/диагностики |
| | `GDB` | отладка приложений (в т.ч. в песочнице) | отладка контейнера/плагинов |
| **Пакеты** | `rpm-validator` | проверка RPM **перед установкой** (валидатор) | `doctor`/CI: валидация артефакта |
| | `rpmsign-external` | подпись установочных пакетов | шаг подписи в `build` |
| **Анализ** | `strace` | трассировка системных вызовов/сигналов | диагностика рантайма |
| | `Svace` | статический анализатор кода | CI-качество нативной части |
| | `Valgrind` | профилирование/поиск утечек | производительность Runtime (NFR-1) |
| **Тестирование** | `testrunner-lite` | автоматизация модульного тестирования | основа conformance/e2e на устройстве |
| **Прочее** | `Git` | контроль версий | базовый инструмент |
| | `MIC` | генерация образов | системные сценарии (вне MVP) |
| | `SSU` | обновление ПО «по воздуху» | вне области Aurobore |

> Также упоминаются **Conan-пакеты** как способ поставки зависимостей — потенциально полезно для
> нативных зависимостей плагинов `(verify)`.

## 2. Что из этого важно для Aurobore

- **`aurobore build`** → `mb2`/`apptool` (сборка) + `rpmsign-external` (подпись) + `rpm-validator` (проверка).
- **`aurobore run`** → деплой/запуск через `ADT`; отладка — `GDB` (учитывая песочницу).
- **`aurobore doctor`** → проверяет наличие SDK, таргетов, прогон `rpm-validator`, типичные проблемы
  (RPATH/зависимости WebView). См. [architecture/cli.md](../architecture/cli.md).
- **CI** → `Svace` (статанализ нативной части), `testrunner-lite` (тесты), `rpm-validator` (валидация).

## 4. Что верифицировать `(verify)`

- Точные команды/флаги `ADT`, `rpm-validator`, `rpmsign-external` для целевой версии SDK.
- Применимость **Conan**-пакетов для нативных зависимостей плагинов.
- Процедура получения **доверенных сертификатов** подписи (через `partners@omp.ru`) и связь с профилями
  безопасности — см. [verification-status.md](verification-status.md).
