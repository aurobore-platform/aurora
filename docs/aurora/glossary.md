# Глоссарий терминов ОС Аврора

> Термины самой **ОС Аврора** и её SDK (дополняет общий [glossary.md](../glossary.md) проекта Aurobore).
> Источники — официальная документация [developer.auroraos.ru](https://developer.auroraos.ru/doc).
> Версия документации: **ОС Аврора 5.2.1**.

| Термин | Определение |
|---|---|
| **ОС Аврора** | Российская мобильная операционная система; целевая платформа Aurobore. |
| **Аврора SDK** | Набор инструментов для **прикладной** разработки: создание, сборка, запуск, тестирование, отладка приложений; включает документацию, примеры, шаблоны. |
| **Platform SDK (PSDK)** | Набор инструментов для **системной** разработки (компоненты ОС); сборка из командной строки, пригоден для CI/CD. |
| **Аврора IDE** | Среда разработки на базе Qt Creator (C/C++/QML, компоненты Silica). |
| **Aurora OS Build Engine** | Окружение сборки, не зависящее от хостовой ОС (вариант поставки SDK с `mb2`); виртуализация — Docker. |
| **Аврора Build Tools (BT)** | Вариант поставки SDK с кросс-компиляцией (инструмент `apptool`, sysroot для armv7hl/aarch64). |
| **`mb2`** | Инструмент сборки в варианте Build Engine. |
| **`apptool`** | Инструмент сборки в варианте Build Tools. |
| **`sb2`** | Инструмент компиляции/запуска кода в окружении таргета (Platform SDK). |
| **`ADT`** | Консольная утилита для отладки, управления и взаимодействия с ОС Аврора. |
| **`rpm-validator`** | Инструмент проверки RPM-пакетов перед установкой в ОС Аврора. |
| **`Svace`** | Статический анализатор кода (поиск ошибок/аномалий). |
| **`Valgrind`** | Инструмент профилирования и поиска утечек. |
| **`strace`** | Утилита трассировки системных вызовов и сигналов. |
| **`testrunner-lite`** | Инструмент автоматизации модульного тестирования. |
| **`MIC`** | Средство формирования образов (генерация локального образа). |
| **`SSU`** | Сервис обновления ПО «по воздуху» (OTA). |
| **Conan** | Менеджер C/C++-зависимостей; в экосистеме Аврора есть Conan-пакеты. |
| **Aurora OS Emulator** | Виртуальная машина (QEMU), исполняющая приложения в окружении ОС Аврора. |
| **МУ** | Мобильное устройство — реальное устройство под управлением ОС Аврора. |
| **Таргет (target)** | Окружение сборки под конкретную ОС/архитектуру; имя вида `AuroraOS-<релиз>-base-<arch>`. |
| **Архитектуры** | `x86_64` (эмулятор), `armv7hl` (32-бит ARM), `aarch64` (64-бит ARM). |
| **sysroot** | Набор библиотек и исходников для кросс-компиляции под целевую архитектуру (в поставке BT). |
| **Кросс-компиляция** | Сборка под архитектуру, отличную от хостовой (через `gcc`/`sb2`). |
| **RPM** | Формат установочного пакета приложения/компонента; перед установкой на устройство **подписывается**. |
| **`.spec`** | Спецификация RPM-пакета (имя, версия, зависимости, файлы). |
| **`.desktop`** | Файл точки входа приложения; содержит метаданные запуска и **декларацию разрешений** и D-Bus-службы. |
| **`rpmsign-external`** | Утилита подписи RPM ключом/сертификатами. |
| **`pkcon install-local`** | Установка локального RPM-пакета на устройстве. |
| **Песочница (изоляция)** | Изолированное окружение исполнения приложения (через **Firejail**), ограничивающее доступ к API и данным. |
| **Firejail** | Оболочка изоляции, дающая процессу собственное представление ресурсов ядра. |
| **Общие директории** | Механизм обмена данными между изолированными приложениями (вместо прямого доступа к чужим файлам). |
| **Профиль безопасности** | Набор ограничений, выбираемый автоматически при валидации пакета по **типу сертификата подписи**. |
| **`OrganizationName` / `ApplicationName`** | Значения, на основе которых приложению выдаются права на имя D-Bus-службы на сессионной шине. |
| **`runtime-manager-tool`** | Утилита для отладки приложений, запущенных в песочнице (альтернатива `gdb`). |
| **Silica** | Набор UI-компонентов для построения интерфейса приложений ОС Аврора. |
| **WebView (фреймворк)** | Встраиваемый веб-компонент; позволяет адаптировать веб-приложение, встраивая его в нативное приложение (QML/C++). |
| **PWA** | Прогрессивные веб-приложения; отдельный путь доставки (в т.ч. установка из браузера), смежный с WebView. |
| **Flutter (на Авроре)** | Поддерживаемый стек с адаптированными средой, плагинами и примерами; код исполняется без изменений. |

> Примечание: значения вида `ru.auroraos.*` — пространство имён пакетов в примерах документации;
> точные имена конкретных пакетов (например, WebView) подтверждаются против установленного SDK (`verify`).
