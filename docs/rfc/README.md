# RFC (Request for Comments)

RFC — площадка для **предложений, ещё не принятых** как решения. Здесь обсуждаются крупные изменения,
новые подсистемы или альтернативы текущей архитектуре до того, как они станут [ADR](../adr/README.md).

## Когда писать RFC
- Изменение публичного контракта (протокол моста, формат манифеста/конфига, API SDK).
- Новая крупная подсистема (например, кросс-платформенный режим, бинарный протокол).
- Спорный выбор с несколькими равноценными вариантами.

## Процесс
1. Создать `RFC-XXX-краткое-имя.md` по [шаблону](RFC-000-template.md), статус `Draft`.
2. Обсуждение (issues/PR), сбор обратной связи → статус `Review`.
3. Решение: `Accepted` → оформляется соответствующий **ADR**; либо `Rejected`/`Withdrawn` с причиной.
