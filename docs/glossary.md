# Глоссарий

Единый язык проекта. Термины упорядочены по алфавиту (латиница, затем кириллица).

| Термин | Определение |
|---|---|
| **App (приложение)** | Веб-приложение пользователя (HTML/CSS/JS/TS), упакованное Aurobore в нативное приложение Аврора. |
| **Asset Loader** | Подсистема Runtime, отдающая локальные веб-ресурсы в WebView через защищённую кастомную схему (а не `file://`). |
| **Bridge (мост)** | Канал двунаправленной связи между JS и нативным кодом: вызовы методов, ответы (Promise), события, стримы, ошибки. Состоит из JS-стороны и native-стороны. |
| **Bridge Transport** | Низкоуровневый транспорт моста на `CefMessageRouter`/`cefQuery` (Chromium/CEF). Скрыт за тонким внутренним швом; авторов плагинов не касается. |
| **CefMessageRouter / cefQuery** | Штатный асинхронный мост CEF: JS `window.cefQuery({request,onSuccess,onFailure})` ↔ C++ `Handler::OnQuery(...callback)`. Основа Bridge на Авроре. |
| **Aurobore** | Рабочее кодовое имя платформы. |
| **Capability / Scope (область)** | Ограничение, какие плагины и какие ресурсы доступны приложению (например, FS только в каталоге данных приложения). |
| **CLI** | Инструмент командной строки Aurobore (`aurobore`, алиас `bor`): create / dev / build / run / plugin / doctor / publish и др. |
| **Codegen (кодогенерация)** | Генерация JS-обёрток, TypeScript-типов и native-«заглушек» регистрации из манифеста плагина. |
| **Config (`aurobore.config`)** | Файл конфигурации проекта: appId, имя, версия, ориентация, разрешения, плагины, настройки сборки. |
| **DevTools** | Встроенные средства разработчика: лог моста, инспекция вызовов, профилирование. |
| **Dev Server** | Локальный HTTP-сервер для разработки; WebView грузит приложение с него для быстрой итерации. |
| **Deep Link** | Переход в приложение по URI/схеме (например, из другого приложения или уведомления). |
| **Engine (движок WebView)** | Браузерный движок внутри WebView Аврора. Целевой — **Chromium/CEF** (`ru.auroraos.webview`). Gecko (легаси, до 5.1.3) вне поддержки. |
| **Event (событие)** | Однонаправленное «выстрелил-и-забыл» сообщение (lifecycle, изменения состояния). Может идти из натива в JS и обратно. |
| **Hot Reload** | Обновление модулей приложения без полной перезагрузки и без потери состояния (через HMR веб-сборщика). |
| **Invoke (вызов)** | Запрос из JS к нативному методу плагина с аргументами и ожиданием ответа (Promise). |
| **Live Reload** | Полная перезагрузка WebView при изменении исходников (более грубо, чем Hot Reload). |
| **Manifest (манифест плагина)** | Декларативное описание плагина: имя, версия, методы, события, типы, разрешения, нативные зависимости. Единый источник истины. |
| **Native SDK** | C++/QML библиотека и контракты для авторов нативных плагинов. |
| **Permission (разрешение)** | Право приложения на доступ к ресурсу системы (камера, геолокация, интернет и т.д.); проецируется в артефакты Аврора (`.desktop`). |
| **Plugin (плагин)** | Модуль, добавляющий нативную возможность: манифест + нативная реализация + (сгенерированная) JS-обёртка + типы. |
| **Plugin Loader** | Подсистема Runtime, обнаруживающая установленные плагины и регистрирующая их в Bridge/Plugin Manager при старте. |
| **Plugin Manager** | Реестр плагинов в Runtime; маршрутизирует вызовы и события к нужному плагину. |
| **Promise API** | Основная модель ответа моста: каждый `invoke` возвращает Promise. |
| **RPM / `.spec`** | Формат пакета и его спецификация в Аврора; результат сборки приложения. |
| **Runtime** | Нативный контейнер (C++/QML), запускающий WebView и обслуживающий мост, lifecycle, навигацию, ресурсы, разрешения, deep links. |
| **Stream (стрим)** | Многократная доставка данных по одной подписке (геолокация, сенсоры, прогресс). Поверх событий с корреляцией. |
| **Template (шаблон)** | Заготовка проекта (vanilla / react / vue / svelte и т.д.), используемая `create`. |
| **TypeScript SDK** | Публичные npm-пакеты `@aurobore/*` — типизированный API к Runtime и плагинам. |
| **WebView** | Встраиваемый веб-компонент Аврора (Chromium/CEF, `ru.auroraos.webview`), в котором исполняется веб-приложение. |
| **Аврора SDK** | Официальный набор инструментов для сборки приложений Аврора; Aurobore генерирует под него проект и вызывает штатную сборку. |
| **Манифест проекта** | См. *Config*. |
