# Туториалы (план)

> На этапе проектирования здесь зафиксирован **план** практических руководств. Сами туториалы пишутся
> по мере появления соответствующей функциональности (см. [roadmap.md](../roadmap.md)).

Туториалы — пошаговые практические руководства (в отличие от справочной/архитектурной документации).
Каждый туториал ориентирован на конкретный результат и проверяется на реальном проекте.

## Запланированные туториалы

| Туториал | Что освоит читатель | Появится на этапе |
|---|---|---|
| **Быстрый старт** | Установить CLI, `create`, `dev`, увидеть приложение на эмуляторе | MVP |
| **Первое приложение** | Структура проекта, `aurobore.config`, навигация SPA, lifecycle-события | MVP |
| **Сборка и запуск на устройстве** | `build` → `.rpm`, `run`, разрешения, `doctor` | MVP |
| **Использование плагинов** | Добавить плагин (`plugin add`), вызвать метод, обработать ошибки | MVP |
| **Работа с событиями и стримами** | Подписки, геолокация/сенсоры, отписка | Alpha |
| **Написание своего плагина** | Манифест → native → кодоген → conformance-тест | Alpha |
| **Шаблоны фреймворков** | React/Vue/Svelte + адаптеры SDK | Alpha |
| **Отладка через DevTools** | Лог bridge-вызовов, инспекция, профилирование | Beta |
| **Публикация** | `publish`: приложение в стор Аврора, плагин в npm | RC |
| **Миграция между версиями** | `migrate`, обновление конфига/манифеста | Post-1.0 |

## Принципы туториалов
- Один результат на туториал; работающий пример в `examples/`.
- Минимум теории — ссылки на архитектурные документы для деталей.
- Проверяемость: шаги воспроизводятся на чистом окружении (учитывается в e2e-тестах).
